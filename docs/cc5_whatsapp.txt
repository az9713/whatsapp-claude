  ---
  name: schedule-job
  description: Schedule tasks using natural language time expressions
  allowed_tools:
    - Bash
    - Read
    - Write
  ---

  # Schedule Job Skill

  Parse natural language scheduling requests and create cron jobs.

  ## Time Parsing Examples
  - "at 15:46" → minute=46, hour=15
  - "every morning at 9am" → 0 9 * * *
  - "every monday at 8am" → 0 8 * * 1
  - "every night at midnight" → 0 0 * * *
  - "in 30 minutes" → calculate from current time

  ## Implementation
  1. Parse the time expression from user input
  2. Parse the task description
  3. Generate cron expression
  4. Create cron entry:
     {minute} {hour} {day} {month} {weekday} cd {WORKSPACE} && claude -p "{task}"
  --dangerously-skip-permissions >> /var/log/agent-cron.log 2>&1
  5. Add to crontab using: `(crontab -l; echo "{entry}") | crontab -`
  6. Confirm job was scheduled

  ## List Jobs
  - Run `crontab -l` to show all scheduled jobs

  ## Remove Job
  - Parse which job to remove
  - Use `crontab -l | grep -v "{pattern}" | crontab -`

  Why 65% confidence: Time parsing is the tricky part. Simple times like "at 15:46" are easy.
  Complex expressions like "every other tuesday" need more sophisticated parsing. Could use a
  library or rely on Claude's reasoning.
